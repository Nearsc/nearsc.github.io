CICE 使用手册

发表于 2017-08-23   |   分类于 教程   |  

在学习模式以及调试模式的过程中，我不太习惯把每个步骤或细节都记录下来，也懒得把一些关于模式的有用信息整理成文档。我自己要查看某些信息的时候，就直接去源代码里看了。话虽如此，整理工作还是很有必要的。这里就先写一份海冰模式 CICE 的使用手册，以供师弟师妹们查阅。

更多信息请参考 CICE 官网。

快速入门

准备工作

正式开始之前，我们需要做好两方面的准备工作，即下载模式代码和模式数据，并将它们上传到服务器。在我们服务器上，我已经把代码和数据放到了相应的目录下：

  1. 模式代码

    /home/public/practice/cice/cice.v5.0

  2. 模式数据

    /home/data/model-forcing/cice

模式代码目录下有两个文件夹：Original 和 test。其中 Original 下存放的是网上的原始代码，而 test 下存放的是我修改过的能够快速使用的代码。test 中所做的修改是关于环境配置以及模式配置的，可以之后再慢慢去了解。
我们需要做的是，在自己的帐号目录下建立一个存放模式试验的文件夹，然后将模式代码拷贝到那个文件夹下。以我的帐号为例：

在我建立的 test1 目录下，有一个 src 文件夹和一个 readme.txt 文件。源代码即存放在 src 里，而 readme.txt 则用来记录该试验（test1）的相关信息。如果你要做另一个试验，并创建了文件夹 test2，那么就可以修改 test2/readme.txt 里的内容，以记录 test2 所作的改动。这样的做法有利于之后快速区分不同试验间的主要差异。
模式数据不需要拷贝，因为数据量较大，如果每个人都把它拷贝到自己的帐号目录里，就会占用过多的硬盘空间。实际上我们就把它放在 /home/data/model-forcing/cice 里，然后都读取那个目录下的文件就行了。
P. S.：一定要记着把 /home/public/practice/cice/cice.v5.0/test 拷贝成新的文件夹（如test1），不要直接对那个公共目录进行修改。当然，你也没有权限修改它。

模式配置文件

comp_ice

ice_in

CICE 的大部分模式配置都在 ice_in 文件里进行，该文件的存放位置为 $SYSTEM_USERDIR/src/input_templates/$RES/ice_in。这里 $SYSTEM_USERDIR 即为前文中提到过的模式根目录，而 $RES 则应随分辨率的改变而取不同的值。实际上根据分辨率的不同，模式需使用不同的 ice_in 文件，这些文件被分别放在了以分辨率为名的各个文件夹下。根据 $RES 所取的不同的值，comp_ice 在编译模式的同时，会将相应的 $SYSTEM_USERDIR/src/input_templates/$RES/ice_in 文件拷贝到 $SYSTEM_USERDIR/rundir 下。这意味着，你可以在编译前先修改 $SYSTEM_USERDIR/src/input_templates/$RES/ice_in，也可以编译后直接修改 $SYSTEM_USERDIR/rundir/ice_in。我的建议是采用前者的方式，因为这样可以对修改进行存档，即使你把 rundir 删掉了，你所做的修改也仍然放在 $SYSTEM_USERDIR/src/input_templates/$RES 下。但需要记住 $SYSTEM_USERDIR/rundir/ice_in 才是模式真正读取的文件。

并行计算设置

分辨率设置

分辨率设置其实也就是网格设置，在指定分辨率大小的同时，也要指定每个格点的经纬度、海陆分布等信息。对于 CICE 而言，分辨率的设置必须要在编译模式之前完成。我们先来看看 comp_ice 文件里需要修改的字段：

首先要对 RES 和 GRID 这两个字段进行修改，其中 RES 是某个网格的名称，GRID 为该网格的分辨率大小。实际上网格名称是具有一定含义的，如 gx1 和 gx3 意指 1°×1° 和 3°×3° 的网格。这里采用的是 gx1 网格，如要换成 gx3，就把 gx1 那行注释掉，然后把 gx3 那行取消注释就行了。
之后还需要对 BLCKX 和 BLCKY 字段进行修改，即改变块的大小。由于整个网格是被划分为若干块并由多个 CPU 并行计算的，若分辨率改变，就要同时改变每一小块的大小，以保证分辨率、核数、块大小满足整除的关系。具体细节可查看 并行计算设置。
由于 comp_ice 会根据不同的 RES，自动读取对应的 $SYSTEM_USERDIR/src/input_templates/$RES 下的网格信息文件，因此设置完 RES 后，CICE 就能自动地获取到所需的经纬度、海陆分布信息了。而我们需要做的，是修改 ice_in 文件里的内容，这牵涉到两个方面：

  1. 将已做的参数修改进行“搬家”。假如你之前使用的是 gx1 网格，并在这个网格下做了一些试验，修改了一些参数（如修改时间），那么你修改的是 $SYSTEM_USERDIR/src/input_templates/gx1/ice_in。之后你想使用同样的参数，但换成 gx3 网格，你会发现 gx3/ice_in 里的参数还是原始设置。这时候你就需要把 gx1/ice_in 里的修改挪到 gx3/ice_in 里去。

强迫场设置

时间设置

CICE 的时间设置主要涉及到 ice_in 文件里的 year_init、dt、npt、fyear_init、ycycle 参数：

year_init 为模式的起始年份，dt 为时间步长（秒），npt 为模式运行的总步数。若要改变模式运行时间，通常先修改模式开始的年份 year_init；然后根据你要运行的总时间以及 dt，计算得到模式运行的总步数 npt，如 31 年即 977616000 秒，则 npt=977616000/3600=271560。一般不对时间步长 dt 进行修改。
