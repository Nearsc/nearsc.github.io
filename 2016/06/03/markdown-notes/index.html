Markdown 备忘录

发表于 2016-06-03   |   分类于 备忘录   |  

终于决定要开始整整我的主页了，然后接触到了Markdown，就考虑用它来完成我大部分blog的撰写。虽说使用起来简易方便，没有什么难记的语法，但我感觉自己忘性比较大，还是写个备忘录来记录一下吧。

基本语法

Markdown最为基础的一些语法，不包括那些扩展的功能。

标题

使用 # 将其设置为标题

# 基本语法
## 标题

强调字体

前后使用 ** 或 __ 使字体变为粗体。
前后使用 * 或 _ 使字体变为粗体。
前后使用 ** 或 __ 并且前后使用 * 或 _ 使字体变为粗斜体。
前后使用 ~~ 使字体被划掉。

**我是粗体**
__我也是粗体__
*我是斜体*
_我也是斜体_
**_我是粗斜体_**
__*我也是粗斜体*__
~~我被划掉了~~

我是粗体
我也是粗体
我是斜体
我也是斜体
我是粗斜体
我也是粗斜体
我被划掉了

引用文字

使用 > 将其设置为引用。当然，可以不是名人名言。

>Remember the past, know the future.

Remember the past, know the future.

代码块

使用4个空格或者1个tab来获得代码块

<Tab>printf("这是一个代码块")

printf("这是一个代码块")

链接

使用 []() 来添链加接

详情请访问[Nearsc的主页](http://nearsc.github.io/)

详情请访问Nearsc的主页

图片

使用 ![]() 来添加图片

![罗小黑](/images/lxh.gif)

项目清单

使用数字来添加项目编号
使用 - 或 * 或 + 来添加项目符号
使用两个空格获得嵌套清单

GitHub 特殊语法

包含了部分的 GFM (GitHub Flavored Markdown)

引用代码

使用 ` 来加入一串代码
嵌入式代码 —— 前后各一个`

`tar`是Linux下用来打包的命令

tar是Linux下用来打包的命令

代码块 —— 前后连续三个`

```bash
cd ~/Download
tar -xvzf netcdf-4.4.0
cd netcdf-4.4.0
mkdir -p /opt/netcdf/netcdf-4.4.0
./configure --prefix=/opt/netcdf/netcdf-4.4.0
make all install
```

任务清单

使用 - 和 [ ] 标记为未完成的任务
使用 - 和 [x] 标记为已完成的任务

- [x] 给植物浇水
- [ ] 收集雨花石
- [ ] 学画画

使用 - 和 | 来绘制表格
